# CodexMeter 设置面板精简设计

## 目标

采用 macOS 系统设置风格，重新组织 CodexMeter 设置窗口。此次调整以移动位置、简化层级和统一样式为主，保留现有功能与用户配置；仅移除界面中重复或不具备可操作性的实现信息。

## 设计原则

1. 设置入口保持稳定且容易定位，不把菜单栏、下拉面板和小组件合并成二级切换。
2. 常用选项直接展示，低频微调通过原生折叠组或条件内容收纳。
3. 沿用现有 `@AppStorage`、偏好键、通知和默认值，不迁移或删除已保存配置。
4. 优先使用 SwiftUI 原生 `List`、`Form`、`Section`、`DisclosureGroup`、`Toggle` 和 `Picker`，减少自定义卡片与说明控件。
5. 删除独立“高级”入口，将维护和诊断能力放回 Codex 页面。

## 窗口结构

设置窗口采用约 880 × 620 点的可调整窗口，侧栏约 190 点。侧栏使用系统 sidebar 样式，内容区使用 grouped form。移除当前设置内容顶部的应用横幅和账户摘要，为表单释放纵向空间。

侧栏固定保留六个入口：

1. 通用
2. 菜单栏
3. 下拉面板
4. 小组件
5. Codex
6. 关于

侧栏不再重复显示版本号，版本统一在“关于”页面展示。

## 页面内容

### 通用

“系统”分组直接展示：

- 登录时启动
- 刷新频率
- 界面外观

“外观”分组直接展示状态颜色预设。“更多选项”折叠组保留：

- 启动时打开设置
- 卡片不透明度
- 三档自定义状态颜色

### 菜单栏

页面顶部保留实时预览。常用设置直接展示：

- 菜单栏内容
- 显示 5 小时窗口
- 显示 7 天窗口
- 显示 Codex 图标
- 显示活动指示
- 布局模式

布局模式统一为“紧凑 / 标准 / 自定义”。紧凑和标准复用现有预设值；选择自定义时展开以下现有细项：

- 显示百分号
- 活动样式
- 项目间距
- 两行行距
- 数字字号
- 数字字重

工作日刻度属于 Codex 用量计算偏好，移动到 Codex 页面。恢复菜单栏默认操作移动到 Codex 的维护分组。

### 下拉面板

保留所有现有功能，通过业务含义分组，不隐藏可配置项：

- 用量：用量速度、额外额度
- 活动：Profile 概览、Token 活动、额度重置卡
- 洞察：活动洞察、最常用插件
- 雷达：降智雷达；开启后条件展示分值折线图开关
- 显示：同步详情、重置时间样式

### 小组件

保留所有用户可配置项：

- 显示内容
- 显示重置时间
- 显示预期消耗速度
- 显示最近同步
- 显示账户摘要

不再展示“设置变化后自动刷新”和缓存文件名，因为它们是实现说明，不是设置。

### Codex

直接展示连接状态和常用操作：

- 重新读取连接状态
- 打开 Codex 目录
- 打开缓存目录
- 打开 Hook 状态目录

“连接详情”折叠组保留当前数据来源、接口、认证文件和 Hook 路径信息。

“用量计算”分组承载工作日刻度设置。

“维护”分组合并原高级页面的操作：

- 清除最近同步缓存
- 恢复菜单栏默认
- 恢复下拉面板默认
- 恢复小组件默认

### 关于

集中展示：

- 应用图标、名称、简介和版本
- 自动检查更新、更新状态和立即检测
- GitHub、发布记录和问题反馈链接

## 组件与数据流

设置页面继续直接绑定现有 `@AppStorage` 字段，并复用 `appBehaviorBinding`、`surfaceAppearanceBinding`、`menuBarBinding`、`widgetBinding`、`popoverBinding` 和 `codexRadarBinding`。写入后仍由现有通知触发菜单栏、下拉面板、后台刷新与 WidgetKit 时间线更新。

布局预设只写入现有排版字段，不新增偏好键。当前值无法匹配紧凑或标准预设时，布局模式显示为“自定义”。选择紧凑或标准会复用已有预设应用方法；用户调整自定义细项后保持自定义状态。

## 错误与状态反馈

- 登录项操作失败时，继续在对应分组内显示现有错误信息。
- Codex 登录信息缺失时，继续使用当前警告状态，不阻止访问目录和重新读取。
- 缓存清理和恢复操作继续显示当前结果文案。
- Sparkle 更新按钮继续遵循现有可用状态和用户主动检查流程。

## 兼容性

此次调整不删除偏好键、不重置用户设置、不改变默认值语义。低频选项只是移动或折叠；旧版本保存的值打开新设置页后仍能正确显示和生效。

## 验证

1. 构建应用并运行全部现有单元测试。
2. 为布局模式与设置重置范围补充最小的针对性测试。
3. 打包并安装新构建，启动应用确认设置窗口可正常打开和复用。
4. 人工检查六个侧栏入口、折叠组、条件选项、强制明暗外观和滚动布局。
5. 修改各类设置，确认菜单栏、下拉面板、小组件与 Codex 雷达仍按原路径刷新。

## 不在范围内

- 删除或迁移偏好键
- 改变用量计算、接口请求或 Hook 行为
- 新增语言、快捷键、通知等不在本次范围内的功能
- 重做菜单栏、下拉面板或小组件本身的视觉设计
