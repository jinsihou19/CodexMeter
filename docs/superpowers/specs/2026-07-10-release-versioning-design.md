# 发布版本号设计

## 版本格式

- `MARKETING_VERSION` 使用三段语义版本 `MAJOR.MINOR.PATCH`，当前版本为 `1.0.1`。
- `CURRENT_PROJECT_VERSION` 使用正整数构建号，当前构建号为 `2`。
- 应用完整版本为 `1.0.1 (2)`，主应用和 Widget 必须保持一致。
- 设置窗口左侧栏底部常驻显示运行 Bundle 的完整版本。

## 递增规则

- 不兼容变化递增 major；向后兼容功能递增 minor；问题修复递增 patch。
- 同一语义版本可发布多个构建，但每次成功发布必须使用更大的构建号。
- 发布脚本使用工程当前版本打包，成功安装后把工程构建号加一，为下一次发布预留唯一构建号。
- 需要切换语义版本时通过 `CODEX_RELEASE_VERSION=x.y.z` 指定；脚本校验三段数字格式并在成功后同步回工程。

## 产物与保护

- DMG 使用 `CodexMeter-<语义版本>-<构建号>-<架构>.dmg` 命名。
- 任一目标产物已存在时发布立即失败，不覆盖旧版本。
- arm64 与 Intel 包共享同一个语义版本和构建号。
- 发布失败不推进工程版本；全部打包和安装成功后才推进构建号。

## 验证

- 独立测试版本读取、格式校验、下一构建号和工程文件更新。
- 验证主应用与 Widget 的版本设置一致。
- 运行完整单元测试和 Release 构建检查。
